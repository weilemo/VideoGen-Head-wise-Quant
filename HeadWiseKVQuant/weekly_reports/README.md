# Weekly Reports

这个目录用于维护每周实验汇报，方便和老师同步进展、暴露问题、确认下一步方向。

## 命名方式

- 每周一个 Markdown 文档：`YYYY-Www.md`
- 例如：`2026-W20.md`
- 新的一周开始时，复制上一周文档或使用下面的结构新建。

## 推荐结构

```markdown
# YYYY-Www 周报

## 本周目标

- 

## 本周完成

- 

## 关键实验记录

| 实验 | 配置 | 结果 | 结论 |
|------|------|------|------|
|      |      |      |      |

## 主要发现

- 

## 遇到的问题

- 

## 希望老师指导

- 

## 下周计划

- 

## 相关路径

- 代码：
- 结果：
- 日志：
```

## 维护建议

- 实验启动前先写「本周目标」和预期对比。
- 实验完成后及时补充配置、结果路径和一句话结论。
- 每周汇报前重点更新「主要发现」「遇到的问题」「希望老师指导」。
- 和长期状态同步：重要结论继续写入仓库根目录的 `STATUS.md`、`DECISIONS.md` 或 `logs/`。
